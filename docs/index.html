Processing Architecture
GIS Pipeline
Seven-stage asynchronous processing chain running on Celery workers. Each stage is independently logged and can be retried on failure.
01
⬆
Scene Upload
GeoTIFF ingestion via FastAPI streaming endpoint. Up to 500 MB with chunked write.
02
⟳
Reproject
Reproject to UTM Zone 14N EPSG:32614 at 30m native Landsat resolution.
03
✂
Clip & Align
Mask to Dallas AOI bbox. Snap T2 to T1 pixel grid for exact coregistration.
04
≋
Spectral Indices
Compute NDVI, NDWI, MNDWI, NDBI, BSI from six Landsat bands.
05
◈
Classify
Random Forest (200 trees) applied to both scenes. Four output classes.
06
Δ
Change Detect
Pixel-wise T1×10+T2 encoding produces from-to transition matrix and change raster.
07
↓
Persist & Export
Results written to PostGIS. Export GeoTIFF, change PNG, and JSON statistics.

Engineering Stack
Built With Precision

FastAPI + Uvicorn
Async-native Python REST API with Pydantic v2 validation, auto OpenAPI docs, and Prometheus metrics endpoint.

🗺
GDAL · Rasterio
Industry-standard geospatial raster processing. Reproject, clip, align, and normalise Landsat 8/9 multi-band GeoTIFFs.
GDAL 3.6
🌲
Random Forest
200-tree scikit-learn classifier trained on spectral index features. Balanced class weights for minority water class.
scikit-learn 1.4
🐘
PostGIS 3.3
PostgreSQL with spatial extensions. Stores scene extents as POLYGON geometries. Enables spatial intersection queries.
EPSG:32614

Celery + Redis
Async task queue decouples heavy GIS processing from the HTTP lifecycle. Workers process one 500 MB raster at a time.
Celery 5.4
